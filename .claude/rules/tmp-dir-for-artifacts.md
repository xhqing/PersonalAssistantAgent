# 临时产物存放规则

## 适用场景

当需要在项目里生成**临时产物文件**——它们不属于项目本身、不应被 git 跟踪——时,典型包括:

- 生成的图片、LOGO、图标(SVG / PNG 等)
- 预览页、演示 HTML
- 一次性脚本及其输出
- 调试快照、中间数据
- 任何「给用户看一眼 / 临时验证」的文件

## 核心原则

**这类文件一律存放在当前项目根目录下的 `tmp/` 目录里**,不要散落在项目其它位置,更不要放进项目正式目录(如 `src/`、`docs/`、`assets/`)。

原因:

- 避免污染项目结构,临时产物不混入正式代码
- 集中在一个目录,事后一眼可见、一键清理
- 与项目正式内容明确隔离

## 必做前提:加入 .gitignore

**每次新建 `tmp/` 目录时,必须同时确认 `tmp/` 已写入项目的 `.gitignore`**:

- 若项目已有 `.gitignore`,检查是否包含 `tmp/`,没有则追加
- 若项目没有 `.gitignore`,则新建并写入 `tmp/`
- 忽略规则写 `tmp/`(忽略整个目录),不要只忽略特定文件

绝不让临时产物被 git 跟踪、误提交进仓库。

## 操作要点

1. 临时产物路径统一为:`<项目根>/tmp/<子目录或文件>`
2. 可在 `tmp/` 下按用途再分子目录,如 `tmp/logo/`、`tmp/preview/`
3. 产物完成后,在回复里给出 `tmp/...` 的**项目相对路径**,这样在 VSCode 工作区内链接可点
4. 如用户随后要求把某个临时产物保留为正式文件,再按用户指示迁移到项目正式目录
