# 文件操作优先级规则

## 增改查优先，慎用删除

面对任何文件相关任务，优先使用「增、改、查」三种操作：

- **增**：创建新文件或新增内容
- **改**：修改已有文件或更新内容
- **查**：读取、查看文件内容

**尽量不使用删除操作。** 删除文件或目录应作为最后手段，仅在充分评估且无更优替代方案时执行。如需移除内容，优先考虑注释化、标记为弃用或迁移至归档文件等「改」的方式，以降低误删风险并保持可追溯性。

## 操作约束

1. **读取优先**：面对任何文件相关任务，必须先用 Read 工具读取目标文件内容，禁止凭猜测或上下文推断文件状态。
2. **编辑优先于重写**：能用 Edit 修改的就不用 Write 重写整个文件；仅在创建新文件或需要整体重构时使用 Write。
3. **精确匹配**：Edit 操作的 `old_string` 必须与文件实际内容完全一致，包括缩进和换行，否则宁可重新读取再修改。
4. **最小改动**：只改动必要的部分，不连带修改无关内容，保持改动范围可控。
5. **汇报前验证**：任何文件写操作完成后，在向用户汇报前，必须重新读取文件确认修改是否真正生效（详见 `verify-before-report.md`）。
